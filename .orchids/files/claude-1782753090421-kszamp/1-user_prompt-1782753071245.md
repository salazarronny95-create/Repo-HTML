El sistema debe estar diseñado para ser escalable, con una lógica comercial clara y una experiencia de usuario impecable. No entregues solo fragmentos de código; construye la estructura completa del proyecto, configura los archivos necesarios y asegúrate de que el entorno esté listo para ejecutarse.

### 1. Arquitectura y Stack Tecnológico
* **Frontend:** Utiliza un framework moderno y ligero (ej. React con Vite o Vanilla JS estructurado con Web Components, lo que garantice mejor rendimiento). 
* **PWA:** Configura obligatoriamente el `manifest.json` y un `service-worker.js` funcional utilizando Workbox o lógica nativa para el cacheo del App Shell (modo offline básico).
* **Gestión de Datos y Almacenamiento:** Como los archivos serán subidos activamente por los usuarios desde la interfaz, implementa una solución de almacenamiento local robusta para el MVP (como IndexedDB o la API de File System del navegador) o configura un backend ligero en Node.js para guardar los archivos físicos y actualizar dinámicamente el índice de metadatos (id, nombre, fecha_creacion, fecha_subida, responsable, ruta, etiquetas).

### 2. Diseño e Interfaz de Usuario (UX/UI)
* **Framework CSS:** Implementa los lineamientos exactos de Material Design 3 (M3).
* **Responsive Design:** Enfoque estricto en Mobile-First. La interfaz debe adaptarse fluidamente a pantallas de escritorio.
* **Sistema de Diseño:** Aplica estrictamente los colores pasteles de la marca Pronaca para el dashboard. Utiliza estos tonos suaves para los fondos, tarjetas y elementos de interfaz, asegurando un aspecto corporativo, limpio y que evite la fatiga visual.
* **Visualización:** Crea un feed de tarjetas (cards) para los proyectos. Cada tarjeta debe mostrar: Nombre, Fechas, Responsable, y botones de acción (Ver, Descargar).

### 3. Funcionalidades Core (Friendly & Safe)
* **Módulo de Carga Activa por el Usuario (User-Driven Upload):** Desarrolla una zona de carga altamente intuitiva (interfaz interactiva con Drag & Drop y selector de archivos tradicionales) diseñada para que los usuarios finales suban sus proyectos de manera autónoma. 
* **Validación y Seguridad en la Subida:** El sistema debe validar en tiempo real que solo se admitan archivos `.html` (o archivos comprimidos `.zip` que contengan el proyecto). Para garantizar que la carga de un usuario no dañe el repositorio ni sobrescriba el trabajo de otro, implementa un renombrado automático utilizando timestamps únicos y sanitización de nombres de archivos antes de guardarlos.
* **Descarga y Previsualización:** Implementa un visor encapsulado (ej. un `<iframe>` con sandboxing) para previsualizar los archivos HTML cargados sin que afecten los estilos globales de la PWA. El botón de descarga debe permitir al usuario extraer el archivo original intacto.
* **Búsqueda y Filtros:** Construye una barra de búsqueda en tiempo real y chips de filtrado (Por fecha, Por responsable, Orden alfabético) para localizar rápidamente los archivos subidos.

### 4. Instrucciones de Ejecución para el Agente
1. Inicializa el repositorio y la estructura de directorios.
2. Instala las dependencias necesarias.
3. Desarrolla los componentes de la interfaz aplicando la paleta pastel y M3.
4. Implementa la lógica completa para que la subida del usuario guarde el archivo y actualice el feed de proyectos de forma inmediata.
5. Entrega el entorno configurado para que pueda ser ejecutado localmente con un comando estándar (ej. `npm run dev`) y esté listo para ser desplegado.
